# @pulse/web — Pulse of London frontend

React Router v7 (SSR) app on **Cloudflare Workers**. Renders the live anomaly map from `snapshot.json`.

## Local dev

```
pnpm --filter @pulse/web dev       # http://localhost:5173 (workerd; uses the bundled fixture)
pnpm --filter @pulse/web build
pnpm --filter @pulse/web preview   # serve the built worker in workerd (vite preview)
pnpm test                          # unit tests (run from repo root)
pnpm --filter @pulse/web test:e2e  # Playwright smoke (first run: see note below)
```

## Data wiring

- **Dev/test:** no `SNAPSHOT_URL` set — the loader returns the bundled `app/fixtures/sample-snapshot.ts`. The TfL `app_key` never reaches the browser; it lives only in the ingestor.
- **Prod:** the loader reads `SNAPSHOT_URL` from `context.cloudflare.env` (the Worker env) and fetches it server-side for SSR first paint. The client then polls the **same-origin `/api/snapshot`** route (the Worker proxies R2 server-side — no CORS) every 30 s, keeping the last-good snapshot on any failure. On a failed/mismatched SSR fetch the loader falls back to the bundled sample, so the page always renders.
- **Local with live data:** put `SNAPSHOT_URL=https://…/snapshot.json` in `packages/web/.dev.vars` (gitignored); wrangler/vite expose it via `context.cloudflare.env` for `dev`/`preview`.
- **Line geometry:** `public/data/geometry.geojson` is generated by `pnpm bootstrap:geometry` (Plan 1); coordinates are `[lon, lat]` (verified against live data).

## Deploy to Cloudflare Workers

The Cloudflare adapter is already wired: `@cloudflare/vite-plugin`, `workers/app.ts`, `wrangler.jsonc`, and a workerd SSR entry (`app/entry.server.tsx`, `renderToReadableStream`). Two ways to deploy:

**A. CLI**
```
pnpm --filter @pulse/web exec wrangler login   # one-time, browser OAuth
pnpm --filter @pulse/web deploy                # = react-router build && wrangler deploy
```
Set the snapshot URL as a Worker variable (Cloudflare dashboard → your Worker → Settings → Variables), or inline:
```
pnpm --filter @pulse/web exec wrangler deploy --var SNAPSHOT_URL:https://…/snapshot.json
```

**B. Git integration (auto-deploy on push)**
Cloudflare dashboard → Workers & Pages → Create → connect `akmalkhadir/pulse-of-london`. Root directory `packages/web`, build command `pnpm install && pnpm --filter @pulse/web build` (or let it auto-detect React Router), and add `SNAPSHOT_URL` as an environment variable.

**Snapshot needs no CORS:** the browser polls the same-origin `/api/snapshot` route (the Worker fetches R2 server-side), so the snapshot path needs no R2 CORS policy.

**Basemap DOES need bucket CORS:** the `pmtiles` library fetches `basemap/london.pmtiles` from the R2 public URL cross-origin, so the bucket needs a CORS rule allowing `GET` + the `Range` header from the site origin. See `docs/basemap.md`.

## Go-live considerations

- **Hydration timing on "Updated X ago":** the home route computes `now = new Date()` at render. With a live, recently-generated `snapshot.generatedAt`, the SSR vs client label can differ by a second → a React hydration warning. Seed `now` from `generatedAt` and refresh it in a `useEffect`, or add `suppressHydrationWarning` to that `<p>`.
- **Snapshot freshness field:** `freshness.crowdingAgeSec` can be slightly negative because `generatedAt` is stamped at the start of the ~60 s ingest cycle. It's unused by the UI (which derives staleness from `generatedAt`); tidy in the ingestor later if needed.
- **Map live updates preserve pan/zoom:** `MapView` updates sources via `setData()` (no rebuild).
- **Optional caching:** wrap the loader's snapshot `fetch` in the Workers Caches API (~30 s TTL) to cut upstream reads. Correctness holds without it.
- **Playwright:** the first e2e run needs `pnpm --filter @pulse/web exec playwright install chromium`.
